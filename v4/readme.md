# Lernziele

Ziel ist es, Ihre Buildroot + Kernel Konfiguration für das Testen von Treibern vorzubereiten.
Auch soll das Debugger von Anwendungen und Kernel auf dem Device geübt werden.

Als erste Treiber werden Sie einfache virtuelle Treiber-Module erstellen, die sowohl unter Qemu also auch auf dem Raspberry Pi geladen werden können. 

## Folgende Themen werden erarbeitet:

- Vorbereiten des Kernel und RootFS Images für die Treiberentwicklung.
- Treiber Debugging mit Debugger, dmesg, printk usw.
- Treiber Makefile.
- Rudimentäre Treiberfunktionalität.
